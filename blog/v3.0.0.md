# V3.0 版本公告

亲爱的用户，

我们很高兴地宣布 Next Terminal 3.0 版本正式发布！本次更新为了优化资源使用并提高系统稳定性，对免费版本进行了重要调整。

### 免费版本限制调整

自 3.0 版本起，免费版本的使用限制调整如下：

| 限制项 | 数量 |
|--------|------|
| 用户数 | 1    |
| 资产数 | 20   |
| 并发数 | 10   |
| 数据库 | 1 个 |

此次更新还带来了一系列新功能和改进，我们相信这些优化将为您的使用体验带来更多价值。


## 新增功能

### 1. MySQL 数据库代理功能

新增 MySQL 数据库代理功能，提供企业级数据库安全管理能力：

- 支持协议级拦截增删改 SQL 语句
- 支持 SQL 工单发起与审批流程
- 提供完整的数据库操作审计

![database.png](v3_0/database.png)

> 使用方式参考：https://docs.next-terminal.typesafe.cn/usage/database.html

### 2. Web 资产临时白名单

Web 资产新增临时白名单功能，提供更灵活的访问控制：

- 一键将当前 IP 加入白名单
- 支持设置过期时间
- 过期后自动清理，无需手动维护

![temp-white.png](v3_0/temp-white.png)

### 3. HTTPS 双向认证（mTLS）

支持 HTTPS 证书双向认证，大幅提升连接安全性：

- 为每个用户生成独立的客户端证书
- 支持自动生成和吊销客户端证书
- 实现双向加密认证，防止中间人攻击

![mtls.png](v3_0/mtls.png)

> 使用方式参考：https://docs.next-terminal.typesafe.cn/usage/mtls.html


## 安全增强

### 二次认证机制强化

为保护用户账户安全，以下敏感操作现在需要进行二次认证：

- 解绑 OTP（一次性密码）
- 删除 Passkey（通行密钥）
- 使用 Passkey 登录（按需）
- 修改系统设置（按需）

### SSH 私钥安全

- SSH 代理服务器认证私钥不再回显，防止敏感信息泄露

## 问题修复

- 修复菜单宽度显示不正确的问题
- 修复会话分享强制要求登录的问题

## 功能优化

### 资产管理优化

- **资产树搜索**：支持空字符串搜索（刷新功能）
- **资产别名**：支持为资产设置别名，SSH 直连时可使用别名作为目标名称
- **IP 快速复制**：终端新增一键复制 IP 地址功能

### 访问控制优化

- **地区访问限制**：完善 Web 资产的地区访问限制功能，支持更精细的访问控制

### SSH 功能增强

- **代理服务器认证**：SSH 代理服务器新增手动输入账号密码功能，提供更灵活的认证方式
